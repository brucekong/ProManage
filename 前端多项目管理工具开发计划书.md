# 前端多项目管理工具开发计划书（修订版）

# 一、项目概述

## 1.1 项目背景

当前前端开发过程中，存在多项目并行开发的场景，每个项目（含前后端单仓项目）均需单独执行启动命令（如 npm run dev），存在以下核心痛点：终端多窗口切换繁琐、项目运行状态无法全局可视化、端口冲突难以快速排查、项目启停操作分散、进程残留导致资源浪费。现有终端工具无法满足"全局掌控、一键操作"的需求。

## 1.2 项目目标

- **核心目标**：基于 Tauri + Rust + Vue3 开发一款 Mac 桌面端应用（DMG），实现多项目状态全局可视化、一键启停、端口冲突自动处理、进程管理等核心功能。同时提供 CLI 命令行工具作为轻量补充。

- **架构简化**：放弃"终端 + Web 面板 + 桌面端"三界面架构，聚焦 **Tauri 桌面端为主、CLI 为辅**，避免三套交互层的维护负担。全部后端逻辑用 Rust 实现，不依赖 Node.js 运行时，单二进制分发。

- **短期目标（1-2周）**：Tauri 桌面端基础功能（项目扫描、状态查看、一键启停、端口检测），CLI 基础命令可用。

- **长期目标（3-4周）**：项目分组、启动依赖链、资源监控、系统托盘、日志面板、快捷键、深色模式、DMG 打包分发。

## 1.3 适用范围

适用于 Mac 端前端多项目并行开发场景，支持纯前端项目、前后端单仓项目，兼容 npm、yarn、pnpm 等包管理工具，适配 npm workspaces / pnpm workspaces / turborepo 等 monorepo 方案。

# 二、需求分析

## 2.1 核心功能需求

| 需求模块 | 具体需求描述 | 优先级 |
|---|---|---|

| Monorepo 智能识别 | 1. 自动检测 pnpm workspace / npm workspace / turborepo / lerna 配置；2. 解析 workspace 下的子包并展平展示；3. 支持过滤 workspace 中的非前端项目 | 高 |
| 状态可视化 | 1. 全局展示所有项目运行状态（运行中/已停止/异常）；2. 显示项目端口、PID、启动时间、资源占用；3. 不同颜色标签区分状态 | 高 |
| 一键操作 | 1. 一键启动/关闭所有项目；2. 单个项目启动/关闭/重启；3. 支持系统托盘图标快速操作 | 高 |
| 启动依赖链 | 1. 可配置项目启动顺序（如先后端再前端）；2. 支持健康检查（端口检测 / HTTP 探测），前序项目就绪后再启动下游项目；3. 依赖启动失败时中止后续启动并告警 | 高 |

## 2.2 进阶功能

- **项目分组**：按业务线/项目类型分组管理，支持批量操作某一组
- **场景预设**：保存常用项目组合为"场景"，如"日常开发 = 后端API + 前端PC + 前端H5"，一键启动整套环境
- **环境变量管理**：为每个项目配置独立的环境变量，不同场景切换不同环境配置
- **资源监控**：显示每个子进程的 CPU/内存占用，异常高时告警
- **Git 信息集成**：显示项目当前 Git 分支、未提交修改数量
- **终端输出实时查看**：在桌面端面板实时 tail 项目 stdout/stderr
- **开机自启**：登录时自动启动并恢复上次运行的项目组合
- **深色/浅色模式**：跟随系统主题自动切换
- **配置导出/导入**：多设备复用配置

## 2.3 非功能需求

- **易用性**：界面简洁清晰，无需复杂配置即可上手，拖拽添加项目
- **性能**：项目扫描、状态更新延迟 ≤1s，应用启动快（＜2s），内存占用低（＜100MB）
- **兼容性**：适配 MacOS 12+，支持 M 系列与 Intel 芯片
- **稳定性**：长期运行无崩溃，进程管理无残留，端口检测准确，异常时自动清理
- **原生体验**：Tauri 打包体积小（10-20MB），启动快，支持 Mac 原生交互

# 三、技术方案

## 3.1 技术栈选择

| 技术模块 | 技术选择 | 选择理由 |
|---|---|---|
| 桌面端框架 | [Tauri 2.x](https://v2.tauri.app/) | 轻量高性能，打包体积小，原生体验好，支持 Mac 系统特性 |
| 后端语言 | Rust | 性能优异、安全性高，单二进制分发，无需额外运行时 |
| 前端框架 | Vue 3 + Vite + TypeScript | 开发效率高，生态成熟，与 Tauri 配合良好 |
| 进程管理 | `std::process` + [`sysinfo`](https://crates.io/crates/sysinfo) | 标准库管理子进程，sysinfo 监控系统进程状态 |
| 端口管理 | [`portpicker`](https://crates.io/crates/portpicker) | 查找空闲端口，轻量可靠 |
| 文件操作 | `std::fs` + [`serde`](https://crates.io/crates/serde) | 处理 JSON 配置文件、日志文件 |
| 日志记录 | [`tracing`](https://crates.io/crates/tracing) | 结构化日志，支持分级、文件输出 |
| 系统通知 | `tauri-plugin-notification` | Mac 原生系统通知 |
| 窗口控制 | `tauri-plugin-shell` + Tauri API | Mac 原生窗口、系统托盘、快捷键 |
| 异步运行时 | [`tokio`](https://crates.io/crates/tokio) | 高效的异步 I/O，用于进程监控、端口检测 |
| 终端输出 | [`xterm.js`](https://xtermjs.org/) | 在桌面端嵌入终端，实时展示子进程输出 |

## 3.3 核心功能实现方案

### 3.3.1 项目扫描与 Monorepo 支持

1. 用户指定根目录，Rust 递归遍历子目录查找 `package.json`
2. 对每个 `package.json`，检查 `workspaces` 字段（pnpm / npm / yarn），解析 workspace 指向的实际子包路径并展平
3. 提取项目 `name`、`scripts`（默认 `dev`）、检测 `framework`（vite / webpack / vue-cli）
4. 结果缓存到 `projects.json`，监听文件变更自动更新

### 3.3.2 进程管理与启动依赖链

1. 通过 `std::process::Command` 启动子进程，捕获 stdout/stderr，记录 PID
2. 通过 `sysinfo` crate 轮询进程存活状态（~1s 间隔），异常退出时触发告警
3. **启动依赖链**：
   - 用户可拖拽排序或配置依赖关系（如 B 依赖 A）
   - 启动 B 前，先启动 A 并检测 A 的端口/HTTP 响应，就绪后再启动 B
   - 依赖启动超时或失败时，中止下游启动并发送系统通知
4. 应用退出时，`Drop` 所有子进程句柄，通过进程组杀死全部子进程

### 3.3.3 状态同步机制

- Rust 后端使用 Tauri 的 `AppHandle::emit` 发送事件（`project-updated`、`log-line`、`process-crashed`）
- Vue3 前端通过 `listen()` 监听事件，响应式更新 UI，无需轮询
- 关键事件：进程启动/停止、端口冲突、进程崩溃、日志新行

### 3.3.4 Tauri 系统托盘

- 托盘图标展示运行中项目数量
- 下拉菜单：项目状态概览 → 启停操作 → 显示/隐藏窗口 → 退出应用
- 关闭窗口时自动最小化到托盘（非退出）
