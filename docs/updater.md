# ProStation Remote Update Guide

This project uses the Tauri v2 updater plugin for remote updates.

## What ships to users

- First install: `dmg`
- Later updates: updater artifacts generated by Tauri, plus a signed update feed

The DMG is still useful for first-time installation, but already-installed apps update through the updater feed instead of reinstalling the DMG.

## Current app wiring

The app already supports:

- storing an update feed URL
- storing the updater public key
- checking for updates on launch
- downloading and installing an update
- restarting after installation

Configure those values in the in-app Settings page.

## 1. Generate signing keys

Run this once on a secure machine:

```bash
npm run tauri signer generate -- -w ~/.tauri/prostation.key
```

This creates:

- private key: `~/.tauri/prostation.key`
- public key: `~/.tauri/prostation.key.pub`

Keep the private key secret. The public key goes into the app settings or `tauri.conf.json`.

## 2. Build a signed release

Before building, export the private key and password:

```bash
export TAURI_SIGNING_PRIVATE_KEY="$(cat ~/.tauri/prostation.key)"
export TAURI_SIGNING_PRIVATE_KEY_PASSWORD="your-password"
```

Then build:

```bash
npm run tauri build
```

Because `createUpdaterArtifacts` is enabled in `src-tauri/tauri.conf.json`, the release build will generate updater artifacts in `src-tauri/target/release/bundle/` alongside the DMG.

## 3. Publish the update feed

You can use either:

- a static JSON file
- a dynamic update server

For a simple setup, use a static JSON file.

### Static feed example

See:

- [docs/examples/latest.json](/Users/xuelei.kong/WebstormProjects/ProStation/docs/examples/latest.json)

Host that file at a stable HTTPS URL such as:

```text
https://downloads.example.com/prostation/latest.json
```

Then upload the generated updater artifact files referenced by the JSON.

## 4. Configure the app

In Settings -> Updates:

- `Update feed URL`: your hosted `latest.json`
- `Updater public key`: contents of `~/.tauri/prostation.key.pub`
- `Check for updates on launch`: optional, recommended on

## 5. Test locally

Recommended flow:

1. Build and install `0.1.0`
2. Update the app version to `0.1.1`
3. Build a new signed release
4. Upload the new updater artifacts and refresh `latest.json`
5. Open the installed `0.1.0` app
6. Trigger "Check for Updates" in Settings

If everything is wired correctly, the terminal panel will show an update banner.

## Static JSON format

Tauri's static JSON feed expects:

- `version`
- `platforms.<target>.url`
- `platforms.<target>.signature`

Optional fields include:

- `notes`
- `pub_date`

Common macOS targets:

- `darwin-aarch64`
- `darwin-x86_64`

## Notes

- Production updater endpoints must use HTTPS.
- The signature value must be the text content of the generated `.sig` file, not a file path.
- The public key must be the key content itself, not a path.
- Restart the app after changing updater plugin config.

## GitHub Releases option

This repo includes a starter workflow:

- [.github/workflows/release.yml](/Users/xuelei.kong/WebstormProjects/ProStation/.github/workflows/release.yml)

It uses the official `tauri-apps/tauri-action` to:

- build the macOS app
- upload release assets to GitHub Releases
- upload `latest.json` for the updater

### Required GitHub secrets

Add these repository secrets:

- `TAURI_SIGNING_PRIVATE_KEY`
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

The private key value should be the full contents of your updater private key file.

### Release flow

1. Update app version in:
   - run `npm run release:prepare -- 0.1.1`
2. Commit your changes
3. Push the generated tag:

```bash
git push origin v0.1.1
```

4. Wait for the GitHub Actions workflow to finish
5. Use this endpoint in ProStation Settings:

```text
https://github.com/<owner>/<repo>/releases/latest/download/latest.json
```

6. Paste the matching updater public key in Settings

### Notes for GitHub Releases

- The release tag should match the app version, for example `v0.1.1`.
- The generated `latest.json` is suitable as a static updater feed.
- If you later add Windows or Linux builds, the same feed can include multiple platforms.

## Version bump helper

This repo includes a helper script:

```bash
npm run version:bump -- 0.1.1
```

It updates:

- `package.json`
- `src-tauri/Cargo.toml`
- `src-tauri/tauri.conf.json`

## Release prepare helper

This repo also includes:

```bash
npm run release:prepare -- 0.1.1
```

It will:

- update the app version in all 3 files
- create a local git tag `v0.1.1` when the project is inside a git repository

If the current folder is not a git repository, it will still update the version and tell you the tag command to run later.
